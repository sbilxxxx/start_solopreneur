更新日: 2026-03-20（事例調査を踏まえて改訂）
担当: 広報部エージェント

## 利用規約上の制約（必須）

X Developer Policyにより以下は**規約違反・BAN対象**:
- 自動フォロー・自動アンフォロー
- 自動リプライ・自動DM
- スパム的な大量投稿・同一内容の繰り返し投稿

**許可される自動化:** 自分のコンテンツの自動投稿のみ。
本戦略はすべて規約準拠の範囲で設計する。

---

## ポジショニング

**一言で:**「Claude Code初心者が日本語でソロアントレを実録するアカウント」

**差別化ポイント（事例調査より）:**
- Zennの技術系Claude Code記事は上級者向けが多い → 初心者目線が空白地帯
- 既存のソロアントレ発信者はビジネス系が多い → エンジニアリング解像度が高い点で差別化
- 「稼ぐ目的でClaude Codeを使う実録」という組み合わせは日本語圏でまだ少ない

**読者が得るもの:**
- Claude Codeの実践的な使い方（再現性あり・初心者目線）
- ソロアントレとしての思考・意思決定のリアルな記録
- 失敗・コスト・数字も含めた透明性の高い情報

| 曜日 | 投稿タイプ | 目的 | 時間帯 |
|------|-----------|------|--------|

## コンテンツの柱（4つ）

### 柱1: 学び・気づき（40%）
Claude Code・AI・経営・財務・マーケで得た気づきを短く。
例:「Claude Codeのhooksを使うと、コードを変更するたびに自動でデプロイされる。手動でgit pushしなくなって3日が経つ。」

### 柱2: 実録・作ったもの報告（30%）
サイトの新機能・ブログ記事・仕組みの完成を報告。ブログ記事へのリンク含める。
例:「ブログ機能を実装した。MDXで記事を書けるようになった。[記事URL]」

### 柱3: 失敗・正直な話（20%）
うまくいかなかった話・想定外のコストや時間・率直な感想。
例:「Claude Codeに任せたら思ったより2倍時間がかかった。その理由を整理してみた。」
→ 参考: Sahil Lavingia（Gumroad）は失敗公開で爆発的拡散。「失敗ツイート」は最もエンゲージメントが高い

### 柱4: 数字・進捗（10%）
KPI・コスト・収益など具体的な数字を公開。
例:「このサイトのインフラコスト: ¥0/月。Vercel無料枠で動いている。」
→ 参考: Pat Flynnは収入レポート公開で信頼を構築

## 投稿フォーマット別の使い分け

- **スレッド形式:** Claude Codeの使い方をステップで解説 → 保存・RT狙い
- **数字ツイート:** 「今週◯時間節約した」「PV◯」→ 具体性で共感
- **失敗ツイート:** 「◯◯ができなかった」→ リプライ・エンゲージメント増
- **固定ポスト:** 「誰で何を発信しているか」のまとめスレッド（新規フォロワーへの第一印象設計）

## ハッシュタグ戦略

1投稿につき最大2個まで（固定タグはスパムに見える）。
文脈に合わせて選ぶ:

- `#ClaudeCode` `#AIエンジニア` — engineering記事
- `#ソロアントレ` `#個人事業主` — business記事
- `#個人開発` `#Webサービス` — 作ったもの報告
- `#副業` `#AI活用` — 幅広い層へのリーチ

## 自動化ロードマップ

### 現在（Phase 2）: 半自動
1. ブログ記事を書く
2. `/marketing-post` でSNS投稿文を3パターン生成
3. 人間が確認・選択
4. `scripts/tweet.mjs` で投稿

### Phase 3: 下書き自動生成
- ブログ公開をトリガーにX投稿文を自動生成
- `marketing/sns/drafts/` に保存
- 人間が確認して投稿

### Phase 5: 完全自動化
- X API v2 でスケジュール自動投稿
- エンゲージメント分析 → 次の投稿戦略に反映
