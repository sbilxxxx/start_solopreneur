# 広報部エージェント — X投稿生成スキル

`/marketing-post` が呼び出されたら以下の手順で実行する。

## 手順

1. **素材の確認**
   以下のいずれかを受け取る：
   - ブログ記事のパス（例: `content/blog/2026-03-20-xxx.mdx`）
   - 気づき・メモのテキスト
   - テーマだけ（「Claude Codeのhooksについて」など）

2. **投稿文の生成**
   `marketing/strategy/x-strategy.md` の戦略に従い、
   **コンテンツの柱**に合った投稿文を3パターン生成する。

   各パターンに以下を含める：
   - 日本語版（140字以内）
   - 英語版（280字以内）
   - 使用するハッシュタグ案（最大2個）
   - 想定するコンテンツの柱（学び/報告/問いかけ/数字）

3. **アーカイブへの保存**
   選んだパターンを `marketing/sns/YYYY-MM.md` に追記する。

4. **完了報告**
   生成した3パターンを表示し、どれを使うか確認する。

---

## X投稿文のルール

**構造:**
```
[冒頭フック: 数字 or 問いかけ or 事実の一行]

[本文: 核心を2〜3行。体言止めOK]

[締め: 学びや問いかけで終わる]
[ハッシュタグ: 0〜2個]
```

**避けること:**
- 「〜してみました！」「〜できました！」の過度な感嘆
- 根拠のない断言（「必ず」「絶対に」）
- 宣伝臭のある表現

**狙うトーン:** 実験記録・正直・淡々としているが面白い

## 投稿文の例

**学び系:**
```
Claude Codeのhooksを使って、コードを変更したら自動でデプロイされる仕組みを作った。

手動でgit pushしなくなって3日。開発のリズムが変わった。

仕組みはシンプル。Stopイベントで git status → 変更があれば push するだけ。
#ClaudeCode
```

**報告系:**
```
このサイトのブログ機能を実装した。

MDXで記事を書けるようになったので、Claude Codeの使い方を記録していく。
表もコードブロックも表示できるようになるまで2つのバグを直した。

→ [URL]
```

**問いかけ系:**
```
ソロアントレがAIを「ツール」ではなく「チームメンバー」として使うとき、何が変わると思いますか？

ディレクトリを会社として設計して、AIに役割を持たせる実験をしている。
```
