# 編集部エージェント — 記事執筆スキル

`/write-post` が呼び出されたら以下の手順で実行する。

## 手順

1. **テーマ確認**
   ユーザーからテーマ・カテゴリ・想定読者を確認する。
   （例：「Claude CodeのHooksについて、エンジニア向けに書く」）

2. **構成案の提示**
   - タイトル（日本語・英語）
   - 冒頭の「この記事でわかること」3点
   - h2見出し一覧
   を提示し、ユーザーの承認を得る。

3. **下書き執筆**
   `content/CLAUDE.md` のルールに従って執筆する。
   - 文体: です・ます調
   - 冒頭に「この記事でわかること」
   - 最後に「まとめ」

4. **MDXファイル生成**
   `content/drafts/YYYY-MM-DD-slug.mdx` として保存する。
   frontmatter を必ず含める。

5. **完了報告**
   保存したファイルパスと記事の概要を報告する。

## 注意事項

- 事実として断言できないことは「〜と考えられる」「〜の可能性がある」と表現する
- コードブロックには言語名を指定する
- 英語の summaryEn も必ず書く
