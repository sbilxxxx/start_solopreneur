## ポジショニング（差別化）

### 空白地帯
2026年3月現在、以下の組み合わせで発信している事例はまだ少ない：

> **Claude Code 初心者 × 日本語 × ソロアントレプレナーとして収益化する実録**

技術系ブログの多くは「エンジニアが趣味でClaude Codeを使う話」。
Benのポジションは「コーディング初心者がClaude Codeを武器に一人会社を動かす話」であり、
ターゲット読者に直接刺さる。

### ブランドの一言定義
```
「Claude Code × 一人会社の実験記録」
— AI初心者がソロアントレとして本当に稼げるか、全部見せます
```

## ターゲット読者の定義

| 項目 | 内容 |
|------|------|
| 知識レベル | 大学でコーディングの授業を受けた程度。Web開発は未経験 |
| Claude Code経験 | 名前は知っているか、使い始めたばかり |
| 動機 | 「副業・起業に興味があるが、何から始めるかわからない」 |
| 不安 | 「自分にはスキルが足りない」「失敗したら恥ずかしい」 |
| サイトに来た理由 | X投稿・Zennの記事・Google検索（「Claude Code 初心者」等） |

### 読者の「自分ごと化」ポイント
- 「この人も自分と同じ立場から始めた」と思わせる
- 「自分でもできそう」という感覚を与える
- 失敗談・リアルな数値が読者の安心を生む

## コンテンツ戦略

### 3本柱（コンテンツピラー）

| ピラー | テーマ | 例 |
|--------|--------|-----|
| Claude Code 入門 | 初心者が実際に使った記録 | 「Claude Codeで初めてサイトを作った3日間」「CLAUDE.mdとは何か」 |
| ソロアントレ実録 | 数値・失敗も含めた進捗記録 | 「月次レポート」「最初の1ヶ月でやったこと全部」 |
| 一人会社の作り方 | ツール・仕組み・意思決定の記録 | 「AIで業務設計する方法」「ゼロからXを伸ばした方法」 |

### コンテンツの書き方原則

1. **AI感を排除する** — 「まず〜ます。次に〜ます」調は禁止。自分の言葉で書く
2. **数字を入れる** — 「3日かかった」「コード1行も書かなかった」「PV 0から始めた」
3. **失敗を隠さない** — うまくいかなかった話が最も読まれ、信頼を生む
4. **読者の疑問に答える構成** — 「なぜ？」「どうやって？」「自分でもできる？」の順で書く
5. **タイトルに数字と具体性** — 「Claude Codeで3日間サイトを作った全記録」形式

### 優先的に書くべき記事（次の6本）

1. 「Claude Codeとは何か——コード不要でサイトを作った話」（入門・SEO）
2. 「このサイトをゼロから作った全記録（費用・時間・失敗も全部）」（実録）
3. 「CLAUDE.mdの設計思想——AIを会社の一員にする方法」（入門・差別化）
4. 「ソロアントレ1ヶ月目の正直レポート」（月次・信頼構築）
5. 「Claude Codeで詰まった5つのこと」（失敗談・共感）
6. 「なぜ私はX APIを自動化することにしたのか」（技術 × 戦略）

### クロスポスト戦略
- Zennに技術記事をクロスポスト（技術系読者へのリーチ）
- noteに実録系・ソロアントレ系をクロスポスト（副業・起業層へのリーチ）

## UX設計

### ホームページの構造（改善後）

```
[Hero]
  「Claude Code × 一人会社の実験記録」
  副テキスト: AI初心者がソロアントレとして何ができるか、全部見せます
  CTA: 最新記事を読む（1つだけ）

[About（30秒版）]
  顔写真 or アイコン + 「大学でAIを学び→就職→ソロアントレ始動中。
  Claude Codeだけで会社を動かす実験を記録しています」

[最新記事]
  カテゴリ別 3〜6本

[数字（ちいさくてもいい）]
  投稿◯本 / Xフォロワー◯人 / 運営◯ヶ月目

[CTA（再）]
  Xをフォローする / メルマガ登録（将来）
```

### 記事ページのUX原則
- 冒頭に「この記事でわかること」を3行で書く
- モバイルファースト（Xからの流入はスマートフォンが多い）
- コードブロックはコピーボタンつき
- 記事末尾に「次に読むべき記事」のリンク

### ロードマップのUX改善
- 現状: テキスト中心で読みにくい
- 改善後: フェーズをカード形式で視覚化、「今ここ」を明示
- 読者への伝え方: 「一緒に成長する過程を見てほしい」スタンス

## X（Twitter）マーケ戦略

### 規約上の制約（必須確認済み）
- 自動フォロー・自動リプライ・スパム投稿は規約違反 → BAN対象
- 自分のコンテンツの自動投稿は許可範囲内
- 詳細: X Developer Policy（developer.twitter.com/en/developer-terms/policy）

### 週間ファネル設計（Justin Welsh式の日本語版）

| 曜日 | 投稿タイプ | 目的 |
|------|-----------|------|
| 月 | 気づき・学び（Claude Code / ソロアントレの知見） | 新規フォロワー獲得 |
| 水 | 実録・数値公開（今週やったこと・PV・進捗） | 信頼構築・リピーター化 |
| 金 | ブログ記事・Zenn誘導 | サイト流入獲得 |
| 不定期 | 失敗談・正直な話 | エンゲージメント・拡散 |

### 投稿フォーマット別の使い分け

- **スレッド形式:** Claude Codeの使い方をステップで解説 → 保存・RT狙い
- **数字ツイート:** 「今週◯時間節約した」「PV◯」 → 具体性で共感
- **失敗ツイート:** 「◯◯ができなかった」 → リプライ・エンゲージメント増
- **固定ポスト:** 「誰で何を発信しているか」のまとめスレッド

### ハッシュタグ戦略
メイン: `#ClaudeCode` `#ソロアントレ` `#個人開発`
状況別: `#副業` `#AI活用` `#一人会社`

