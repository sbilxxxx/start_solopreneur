## A. コンテンツ・UX戦略（最優先）

### 問題認識
- 現状のコンテンツはAI生成感が強く、読みづらい
- 読者想定・UX設計が不足している
- ロードマップが直感的に理解しにくい

### ターゲット読者
- 大学でコーディング授業を受けた程度（Web開発は初学者）
- Claude Code・ソロアントレプレナーに興味があるが、まだ始めていない
- 「自分でもできるかも」と思わせることが目標

### 方針
- コンテンツを「自分が初めて読む人として読み直す」基準で全面見直し
- ホームは「このサイトで何が得られるか」を30秒で理解できる設計に
- ロードマップはビジュアルで直感的に把握できるUIに再設計

## C. X運用戦略

### 前提（X Developer利用規約の制約）
- 自動フォロー・自動リプライ・スパム投稿は規約違反 → アカウントBAN対象
- 許可される自動化: 自分のコンテンツの自動投稿のみ
- 詳細: https://developer.twitter.com/en/developer-terms/policy

### 流入獲得の方針（規約準拠）
1. 質の高い投稿コンテンツで自然フォロワーを獲得
2. ハッシュタグ戦略（#ClaudeCode #ソロアントレ #個人開発）
3. ブログ記事公開 → X投稿 → サイト流入の導線設計
4. 手動でのエンゲージメント（返信・RT）は人力で対応

### 自動化スコープ（許可範囲内）
- ブログ公開 → 投稿文ドラフト自動生成（`/marketing-post` スキル）
- 投稿文の人間レビュー → `scripts/tweet.mjs` で投稿
- 投稿結果を `marketing/sns/YYYY-MM.md` にアーカイブ

## D. 収益化戦略（Phase 3〜4 で設計）

### 基本方針
- 情報商材路線ではない（大金を得ることが目的ではない）
- 無料コンテンツを基本とし、一部をプレミアム化
- 知人にはプロモコードで無料アクセス可能な仕組みを作る

### 無料・有料の切り分け案
| コンテンツ | 無料/有料 |
|-----------|---------|
| ブログ記事（入門〜中級） | 無料 |
| 詳細な実装ガイド・テンプレート | 有料 |
| Claude Code スキルファイル集 | 有料 |
| メンバー限定コミュニティ | 有料サブスク |

### 収益化ツール（Phase 6 で実装）
- Stripe（有料記事・サブスクリプション）
- Resend（ニュースレター）
- プロモコード機能

### 参考にする既存事例
- ソロプレナー系ブログ（調査予定）
- Claude Code / AI開発者ブログ（調査予定）

## E. 資金・費用管理（Phase 6 で整備）

### 現状
- X APIクレジット（$5）を個人クレカで購入中
- 事業費と個人費が未分離

### 将来的にやること（収益化が見えてきた段階で）
1. **事業用口座・カードの分離**（住信SBIネット銀行等）
2. **開業届の提出**（個人事業主として経費計上できるようにする）
3. **月次費用トラッキング**（`finance/` ディレクトリで管理）
4. **ROI管理**: 費用 ÷ 新規フォロワー数 / 費用 ÷ サイト流入数
5. **電子帳簿保存法対応**（freee / MoneyForward等）

### 優先しない理由
収益ゼロの段階で管理コストをかけても意味がない。
収益化の目処が立った段階（Phase 6）で一気に整備する。
