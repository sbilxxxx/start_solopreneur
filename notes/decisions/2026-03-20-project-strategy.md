# プロジェクト全体戦略

作成日: 2026-03-20

---

## 何をするプロジェクトか

**一言で:** AIエージェントが各部門を担う「一人会社」をディレクトリとして構築し、その過程でClaude Codeを習得する。

**2つの目的（同時並行）:**
1. 個人ウェブサイト `start-solopreneur.vercel.app` を育てる
2. Claude Code の全機能を実践で習得する

---

## サイトのコンテンツ戦略

| 項目 | 方針 |
|------|------|
| 誰に向けて | AI × ソロアントレを目指すエンジニア・起業家 |
| 何を書く | 「やってみた記録」— 成功・失敗・気づきをそのまま |
| 4つの軸 | Engineering / Business / Finance / Marketing |
| 言語 | 日本語メイン + 英語併記（翻訳はAIが担当） |
| 将来像 | Phase 5以降は記事生成・翻訳・投稿を自動化 |

---

## ディレクトリ＝会社組織

```
start_solopreneur/
│
├── CLAUDE.md          ← 社長指示書（全AIエージェント共通ルール）
├── src/               ← 【開発部】 Webアプリ本体
├── content/           ← 【編集部】 ブログ記事（MDX）
├── marketing/         ← 【広報部】 戦略・SNSテンプレ
├── finance/           ← 【財務部】 収益モデル・試算
├── notes/             ← 【社長メモ】 ← 今ここ
└── .claude/
    └── skills/        ← 各部門エージェントの役割定義
```

**発想の核心:**
- CLAUDE.md（ルート）= 社長から全社員への方針
- CLAUDE.md（各部門）= 部門長への個別指示
- skills/ = 編集者・CFO・マーケターなど各担当AIの定義

## 学習ロードマップ

拡張機能はフェーズではなく **各フェーズの土台** として組み込む。

| Phase | テーマ | 主な習得内容 |
|-------|--------|-------------|
| **1** | 基礎 + CLAUDE.md + Hooks | ファイル操作・デプロイ・永続ルール・自動化の起点 ← **今ここ** |
| **2** | 会社組織化 + Skills + MCP | ディレクトリ設計・エージェント役割定義・外部ツール連携 |
| **3** | 外部API + Subagents | データ収集・KPI可視化・専門ワーカーへの分業 |
| **4** | AI機能実装 | Anthropic SDK・Streaming・RAG検索 |
| **5** | マルチエージェント + Agent Teams | 並列実行・部門間連携・コンテンツ自動生成 |
| **6** | 自動化 + Plugins | フル自律ワークフロー・プラグイン化・Slack通知 |
| **7** | 収益化 | Stripe・メール配信・AIアナリティクス |

## 決定済みの方針

- Webフレームワークは **Next.js**（情報量・将来性）
- デプロイは **Vercel無料枠**（コスト¥0で運用）
- コードを変更したら **自動でGitHub push → 本番反映**
- 記事は **MDXファイル**で管理（コードと分離）
- メモは **このnotesディレクトリ** に日付付きで保存
