## ソロの限界とAIチームの可能性

ソロアントレプレナーの根本的な問題は、**一人でエンジニア・編集者・CFO・マーケターを兼任しなければならない**ことだ。

どれかに集中すれば他が止まる。時間は有限だ。

AIを「便利なツール」として使う限り、この問題は解決しない。AIに「適切な役割」を与えて初めて、一人がチームとして機能し始める。

## ディレクトリを会社として設計する

このプロジェクトのディレクトリ構成はこうなっている：

```
start_solopreneur/
├── CLAUDE.md          ← 社長指示書（全員共通のルール）
├── src/               ← 開発部
├── content/           ← 編集部
│   └── CLAUDE.md      ← 編集部のルール
├── marketing/         ← 広報部
│   └── CLAUDE.md      ← 広報部のルール
├── finance/           ← 財務部
│   └── CLAUDE.md      ← 財務部のルール
└── notes/             ← 社長メモ
```

重要なのは**各部門にCLAUDE.mdがある**ことだ。

## CLAUDE.mdの2つの役割

CLAUDE.mdはClaude Codeが毎回自動で読み込むルールファイルだ。

ルート（`/CLAUDE.md`）は**全社方針**。「npmを使う」「APIキーを直書きしない」といった全員共通のルール。

部門ごとの`CLAUDE.md`は**部門長への指示書**。たとえば`content/CLAUDE.md`にはこう書いてある：

```markdown
## 文体・トーン
- 一人称は「私」
- 語尾は「です・ます」調
- 読者は「AIに興味があるエンジニア・起業家」を想定
```

このファイルがあることで、「記事を書いて」と指示したとき、AIは自動的にこのルールに従って書く。

## Skillsで役割を「呼び出し可能」にする

CLAUDE.mdはルール集だが、**Skills**は「役割そのもの」だ。

`.claude/skills/write-post.md` というファイルを作っておくと、`/write-post` と入力するだけで編集部エージェントが起動し、記事執筆の手順を自律的に進める。

```
/write-post
→ テーマを確認
→ 構成案を提示
→ 本文を執筆
→ MDXファイルとして保存
```

これがClaude Codeの**Skills**という拡張機能だ。

## 「社長として動く」という感覚

このシステムを作ってから、Claude Codeとの関わり方が変わった。

以前は「コードを書いてくれるツール」として使っていた。
今は「各部門の担当者に指示を出す」感覚で使っている。

記事を書くときは編集部エージェントに任せる。
財務を確認するときは財務エージェントに依頼する。
SNS投稿が必要なときは広報エージェントを呼ぶ。

自分がやることは**意思決定と品質確認**だけだ。

## まとめ

- ソロアントレがAIを使いこなすには「役割を与える」発想が必要
- ディレクトリ構成を会社組織として設計することで、AIに部門の文脈を持たせられる
- CLAUDE.mdがルール集、Skillsが役割定義
- 自分は社長として意思決定だけに集中できる
