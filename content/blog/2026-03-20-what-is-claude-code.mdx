このサイト自体、Claude Codeを使って作っている。
コードをほぼ書かずに、会話だけでNext.jsのサイトを構築し、Vercelにデプロイするところまでやった。

この記事では「Claude Codeとは何か」を、使い始めた人間の視点から整理する。

## この記事でわかること

- Claude Codeが従来のAIコーディングツールと何が違うか
- 「会話でコードを書く」とはどういう感覚か
- Claude Codeが持つ主なツールと役割

## Claude Codeは「チャットでコードを書いてもらうもの」ではない

最初の誤解はここだった。

ChatGPTやCopilotは「コードを生成して貼り付ける」ツールだ。人間が最終的にファイルを編集する。

Claude Codeは違う。**ファイルを直接読み書きし、コマンドを実行し、ブラウザで結果を確認するまでを自分でやる。**

つまり、Claude Codeは「コードを提案するAI」ではなく、「実際に手を動かすエンジニア」として動く。

## 何ができるか――主なツール

Claude Codeが持つ主なツールはこれだ：

| ツール | 何をするか |
|--------|-----------|
| `Read` | ファイルを読む |
| `Edit` | ファイルの一部を書き換える |
| `Write` | 新しいファイルを作る |
| `Bash` | ターミナルコマンドを実行する |
| `Glob` | ファイルを検索する |
| `Grep` | コード内のテキストを検索する |
| `WebFetch` | URLの内容を取得する |

これらを組み合わせて、「ページを追加して」「テストを走らせて」「GitHubにpushして」といった指示をそのまま実行する。

## 「会話でコードを書く」の感覚

実際にやってみてわかったのは、これは**「優秀なエンジニアに口頭で指示する」感覚**に近い、ということだ。

```
自分:「ホームページのナビゲーションを共通コンポーネントに切り出して」

Claude Code:
  1. 現在のpage.tsxを読む
  2. components/Nav.tsxを作る
  3. 各ページのナビを差し替える
  4. 動作を確認する
```

自分がやることは「何を作りたいか」を言葉にすることだけだ。
実装の詳細はClaude Codeが判断する。

## 重要な概念：拡張機能

Claude Code単体でも十分強力だが、拡張機能を使うとさらに広がる。

- **CLAUDE.md** — 毎回読み込まれるルール集。「このプロジェクトはnpmを使う」「APIキーは直書きしない」といった方針を書いておくと、毎回説明しなくてよくなる。
- **Hooks** — 特定のイベントで自動実行されるスクリプト。このサイトではClaude Codeが返答するたびに自動でgit pushが走る仕組みを作った。
- **Skills** — 「`/write-post`と入力すると記事執筆モードになる」といった、呼び出し可能なワークフロー。
- **MCP** — GitHubやFigmaなど外部ツールの操作権限をClaudeに渡す仕組み。

## まとめ

- Claude Codeは「コードを提案するツール」ではなく「実際に手を動かすエンジニア」
- ファイルの読み書き・コマンド実行・デプロイまで会話だけで進む
- CLAUDE.md・Hooks・Skills・MCPという拡張機能で自分の開発スタイルに合わせられる
